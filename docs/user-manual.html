User Manual

Application: Spital mobile app

This manual explains how each user role should use the app in daily workflows.

1. First Time Setup

  1. Install the app on your mobile device.
  2. Open the app and choose your language from the language selector.
  3. Create an account or sign in with existing credentials.
  4. If prompted, verify your email using the 6-digit code you receive by email.

2. Login and Account Basics

  • Login: enter email + password on the sign-in screen.
  • Register: choose role, fill required details, and complete email verification.
  • Logout: available from the app bar in each dashboard.
If login fails, verify email/password and check backend connectivity. The app shows a translated error message when credentials are invalid.

3. Role-Based Usage

3.1 Patient

  1. Open Home to see your documents.
  2. Upload a PDF document with the upload button.
  3. To share access with a companion:
    • Generate a 6-digit code, or
    • Send an email invitation token.
  4. Use My Companions to review or remove linked companions.
  5. Open Messages to communicate with the medical side.

3.2 Companion

  1. Sign in with a companion account.
  2. Link yourself to a patient:
    • Redeem a 6-digit code from the patient, or
    • Open and accept an email invitation token/deep-link.
  3. After linking, the patient documents become visible in your home view.
  4. Use My Patients to manage linked patient relationships.

3.3 Doctor

  1. Open the Doctor dashboard.
  2. Use the Patients tab to search/select patient records.
  3. Upload documents to a selected patient.
  4. Use Documents tab to review file list.
  5. Use Messages tab to chat with patient side.
  6. Check ingestion status screen when needed (operational monitoring).

3.4 Hospital Admin

  1. Open Hospital Admin dashboard.
  2. Create/edit/deactivate relevant users in your scope.
  3. Link or unlink patient-companion relationships.
  4. Review hospital-related documents and users.

3.5 Global Admin

  1. Open Global Admin dashboard.
  2. Manage hospitals (create/update/delete).
  3. Manage users across roles and hospitals.
  4. Use role-specific controls to maintain system structure.

4. Companion Access (Detailed)

Numeric Code Flow

  1. Patient generates a code.
  2. Patient shares the code with companion.
  3. Companion enters code in redeem screen.
  4. Link is created if code is valid and not expired.

Email Invitation Flow

  1. Patient sends invitation to companion email.
  2. Companion opens invitation and accepts it.
  3. App links companion to patient and grants view access.
Numeric codes and invitation tokens expire. If expired, ask the patient to generate/send a new one.

5. Messaging

  • Messages are organized by conversation (usually patient context).
  • Unread indicators appear in conversation and dashboard views.
  • Open a conversation, write your message, and send.

6. Language and Translations

  • Language can be changed from the app bar language selector.
  • All UI labels and backend messages are intended to appear translated.
  • If you see a raw key (example: error_invalid_credentials), report it as a translation defect.

7. Troubleshooting

Cannot Sign In

  • Check email/password.
  • Ensure email is verified.
  • Confirm backend is running and reachable.

No Documents Visible

  • For companions: verify patient link is active.
  • Pull to refresh or reopen screen.

Cannot Link Companion

  • Check token/code expiration.
  • Ensure correct role is used (patient generates, companion redeems).

8. Best Practices

  • Use strong passwords and keep credentials private.
  • Only share access codes/tokens with intended recipients.
  • Log out on shared devices.
  • Keep app updated to receive bug fixes and security improvements.

9. Support Checklist

When contacting support, include:

  • your role (patient/companion/doctor/admin)
  • screen where issue occurred
  • exact error message shown
  • time of issue and reproduction steps
